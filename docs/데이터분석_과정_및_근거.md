### 1.2 분석 전략

KEY PAPER(Integrated statistical modeling and machine learning techniques with SHAP for epidemiological data analysis, Elsevier)의 구조를 따라, 전통적 통계 모형(Logistic Regression)과 머신러닝(Random Forest, XGBoost, LightGBM)을 병행하여 이중 검증 구조를 구성하였다.

- **Logistic Regression (statsmodels)**: 계수(Coefficient), 오즈비(Odds Ratio), p-value를 통한 통계적 추론. 변수의 방향성과 유의미성을 확인하며, 계층적 회귀를 통해 변수 그룹별 설명력 변화와 조절효과를 검증한다.
- **머신러닝 (sklearn + imblearn)**: 비선형 관계를 포착할 수 있는 모델로 예측력을 비교하고, 최적 모델에 대해 SHAP을 적용하여 변수 중요도와 조절효과를 비선형적 관점에서 시각화한다.

**LR과 ML의 역할 분담에 대한 이론적 정당화:**

Breiman(2001)은 "Statistical Modeling: The Two Cultures"에서 통계적 모형(추론 중심)과 알고리즘적 모형(예측 중심)이 서로 다른 인식론적 목표를 갖는다고 지적하였다. 이 이분법은 본 연구의 설계 원리와 정확히 부합한다: LR은 변수별 방향성·계수·유의성을 추론하는 **해석 도구**이며, RF/XGBoost/LightGBM은 복잡한 비선형 상호작용 패턴에서 예측력을 극대화하는 **발굴 도구**이다. 두 접근이 동일한 결론에 수렴할 때 결과의 신뢰도는 단일 방법론에 비해 현저히 높아진다(방법론 삼각검증, Methodological Triangulation).

**SHAP의 공리적 토대:**

SHAP(SHapley Additive exPlanations)은 협력 게임이론의 Shapley Value를 기반으로 한다. Lundberg & Lee(2017)는 이 접근이 **효율성(efficiency)·대칭성(symmetry)·허수성(dummy)·가산성(additivity)**의 네 공리를 만족하는 유일한 가법적 특성 중요도 방법임을 증명하였다. 이는 SHAP가 임의적 방법이 아니라 공리적으로 정당화된 해석 기법임을 의미하며, 본 연구에서 SHAP 결과를 LR 계수와 대등하게 해석 근거로 활용하는 이론적 토대이다.

### 1.5 이론적 배경

본 연구의 변수 구성과 결과 해석은 조직안전 이론의 세 가지 축에 의해 정당화된다.

#### 1.5.1 고신뢰조직(HRO) 이론 — 내부 안전관리 변수군의 이론적 기반

Weick, Sutcliffe & Obstfeld(1999)의 High Reliability Organizations(HRO) 이론은 복잡한 고위험 조직이 사고를 억제하는 메커니즘을 다섯 가지 집단적 마음챙김(collective mindfulness) 원칙으로 설명한다: ① 실패에 대한 집착(preoccupation with failure), ② 단순화 거부(reluctance to simplify), ③ 작전에 대한 민감성(sensitivity to operations), ④ 탄력성(commitment to resilience), ⑤ 전문성에 의한 권한 이동(deference to expertise). 이 다섯 원칙은 본 연구의 내부 안전관리 변수군과 직결된다: 안전조직수준·위원회수준은 원칙 ①·⑤, 위험성평가수준은 원칙 ②·③, 작업중지권은 원칙 ④, 정리정돈상태·교육훈련도움·작업반장기여는 원칙 ③의 일상적 작전 민감성을 측정한다.

건설업에서 HRO 원칙 적용 가능성을 실증한 선행연구는 다음과 같다. Zhao et al.(2019)은 중국 건설현장에서 집단적 마음챙김이 안전 풍토를 매개로 사고를 억제함을 구조방정식으로 확인하였으며, Hallowell & Gambatese(2009)는 건설 안전관리 수준 측정 지표가 HRO 원칙과 개념적으로 일치함을 체계적 검토로 제시하였다. 이러한 선행연구를 배경으로, 본 연구는 내부 안전관리 변수의 집합적 효과를 LR의 계층적 회귀 ΔR² 검정과 SHAP 중요도로 실증적으로 평가한다.

#### 1.5.2 스위스 치즈 모델 — 외부감독 변수와 상호작용항의 이론적 위치

Reason(1990, 2000)의 Swiss Cheese Model은 복잡 시스템의 사고 발생을 "다수의 방어층(defence layers)에 뚫린 구멍들이 정렬(align)될 때 사고 경로가 형성된다"는 논리로 설명한다. 각 방어층은 공학적 통제, 절차적 관리, 감독·규제로 구성된다.

이 모델에서 고용노동부 감독은 **최후방 방어층(administrative/regulatory defence)**에 해당한다. 따라서 감독 대상으로 선정된 사업장은 앞쪽 방어층(내부 안전관리, 인증체계)이 이미 충분히 작동하지 못한 현장일 가능성이 높다 — 즉, 감독은 결과(사고 위험의 누적)가 아니라 원인(내부 방어층 취약)에 반응하는 구조이다. 본 연구에서 인증보유 × 고용노동부감독 상호작용항의 OR=2.081이 양(+)의 부호를 갖는 현상은, "인증 취득에도 불구하고 감독이 필요한 현장 = 앞 방어층(인증 이행, 일상적 안전행동)이 이미 불완전하게 작동하고 있는 현장"이라는 **선택편향(selection bias)**으로 이론적으로 방어된다.

#### 1.5.3 사고인과연쇄이론 — 도메인 이론적 배경

Heinrich(1931)의 도미노 이론은 불안전한 행동(unsafe act)과 불안전한 상태(unsafe condition)의 연쇄가 사고로 이어진다는 건설안전의 고전적 인과 모형이다. Bird(1974)는 이를 확장하여 관리적 결함이 연쇄의 기저에 있음을 주장하였다. 이 이론들은 본 연구에서 정리정돈상태(불안전 상태)와 위험성평가수준(관리적 결함 통제)이 종속변수인 사고발생과 갖는 이론적 연관성을 설명하는 보조적 근거로 기능한다.

## 3. Phase 2: 로지스틱 회귀 — 계층적 회귀 분석

### 3.1 분석 방법

연구 설계(RQ)의 논리적 구조에 따라, 변수를 단계적으로 투입하는 계층적(위계적) 로지스틱 회귀분석을 수행하였다. 전체 1,375개 데이터를 대상으로 statsmodels의 `Logit()` 함수를 사용하여 적합하였다.

**순서형 변수 처리에 대한 주석:** 본 연구에서 리커트 척도 변수(안전조직수준·위험성평가수준·교육훈련도움·정리정돈상태·작업중지권·작업반장기여·공사규모·기성공정률 등)는 등간격(equal-interval) 연속형으로 처리하여 로지스틱 회귀에 투입하였다. 이는 사회과학 분야의 일반적 관행이며(Likert 1932; Norman 2010), 특히 5점 이상 척도에서 모수적 처리의 편향이 크지 않다고 알려져 있다. 다만 이 가정이 위배될 경우를 대비하여, 머신러닝(RF/XGBoost/LightGBM)은 분기(split) 기반으로 변수 단위의 가정 없이 비선형 패턴을 포착하므로, LR과 ML 결과의 교차 일치는 등간격 가정의 영향이 제한적임을 간접 지지한다.

계층적 회귀의 목적은 각 변수 그룹이 종속변수(사고발생)의 설명력(Pseudo R²)에 기여하는 정도를 분리하여 확인하고, 최종적으로 조절변수(외부 기관)의 상호작용 효과를 검증하는 것이다.

| 모형 | 투입 변수 | 목적 |
| :--- | :--- | :--- |
| Model 1 | 통제변수 (5개) | 기저 설명력 (현장 특성) |
| Model 2 | + 독립변수 A (3개) | 내부 관리의 추가 설명력 |
| Model 3 | + 독립변수 B (5개) | 현장 안전 행동의 추가 설명력 |
| Model 4 | + 조절변수 (3개) | 주효과 모형 (전체 16개 변수) |
| Model 5 | + 상호작용항 (24개) | 조절효과 검증 (독립 8개 × 조절 3개) |

### 3.2 계층적 회귀 결과

## 4. Phase 3: 머신러닝 모델 비교

### 4.1 클래스 불균형 처리 — SMOTENC

#### 불균형 문제의 진단

종속변수(사고발생)의 분포가 71.6%(0) : 28.4%(1)로 불균형하다. 이 상태에서 머신러닝 모델을 학습시키면, 모든 사례를 '사고 미발생(0)'으로 예측해도 정확도(Accuracy)가 71.6%가 나오므로, 모델이 소수 클래스(사고발생=1)를 전혀 탐지하지 못하는 편향이 발생한다. 실제로 SMOTENC 적용 전 LightGBM의 Recall과 F1이 0.000으로 나온 것이 이를 증명한다.

이 문제를 해결하기 위해 Obasi et al. (2026)이 제시한 리샘플링 전략의 필요성에 근거하여, SMOTENC(Synthetic Minority Over-sampling Technique for Nominal and Continuous features)를 적용하였다(Chawla et al., 2002).

#### SMOTE 대신 SMOTENC를 선택한 근거

본 데이터의 16개 독립변수 중 15개가 정수형(이진, 순서형, 리커트, 범주형)이고, 연속형은 외국인비율 1개뿐이다. 일반 SMOTE는 유클리드 거리 기반 보간(interpolation)으로 합성 데이터를 생성하므로, 이진변수(0/1)에서 0.4 같은 존재하지 않는 값을 만들고, 리커트 5점 척도에서 3.7과 같은 비현실적 값을 생성하는 문제가 있다. SMOTENC는 범주형 변수에 대해 최빈값(mode) 기반으로 합성하므로 이 문제를 회피한다.

범주형 변수 인덱스 설정(15개): 안전조직수준(0), 위원회수준(1), 인증보유(2), 위험성평가수준(3), 교육훈련도움(4), 정리정돈상태(5), 작업중지권(6), 작업반장기여(7), 전문지도(8), 고용노동부감독(9), 안전보건공단지원(10), 공사규모(11), 발주처(12), 기성공정률(13), 공사종류(14). 연속형: 외국인비율(15).

#### 데이터 누수(Data Leakage) 방지 및 Validation Set 처리

SMOTENC를 교차검증 폴드 외부에서 적용하면, 테스트 폴드의 정보가 합성 데이터에 반영되어 성능이 과대평가되는 데이터 누수가 발생한다. 이를 방지하기 위해 `imblearn.pipeline.Pipeline`을 사용하여 **각 교차검증 폴드 내부에서만** SMOTENC가 적용되도록 설계하였다.

```
Pipeline: [SMOTENC → Model]  ← 이 파이프라인 전체가 GridSearchCV의 각 fold에서 독립적으로 실행
```

**핵심: Validation Set에는 오버샘플링을 적용하지 않는다.** 각 5-Fold CV 반복에서 파이프라인은 다음 절차를 따른다:
1. **Training fold (4/5)**: SMOTENC를 적용하여 소수 클래스를 오버샘플링한 뒤 모델 학습
2. **Validation fold (1/5)**: 원본 데이터 분포 그대로 — 오버샘플링 없음 — 예측 후 F1 평가

이 설계가 중요한 이유는, Validation fold가 실제 모집단의 분포(사고 28.4%, 비사고 71.6%)를 그대로 반영해야 교차검증 성능이 배포(deployment) 환경에서의 실제 성능을 편향 없이 추정하기 때문이다. 오버샘플링된 데이터로 Validation을 수행하면 소수 클래스가 인위적으로 늘어나 Recall이 과대평가된다. 최종 Test set(Hold-out 20%) 역시 동일하게 원본 분포를 유지하며 평가한다.

### 4.2 실험 설계

**모델 선택 근거:** 비교 대상으로 Random Forest, XGBoost, LightGBM 세 가지 트리 기반 앙상블 모델을 선택하였다. 이는 다음 이유에 근거한다: (1) **혼합형 변수 처리** — 이진형·순서형·리커트·연속형이 혼재된 본 데이터에서 트리 기반 모델은 변수 스케일링 없이 자연스럽게 분기를 처리한다. (2) **비선형 관계 포착** — 사고발생과 예측변수 간의 비선형·상호작용 패턴을 선형 모델 없이 탐지한다. (3) **SHAP 호환성** — TreeExplainer를 통한 정확한 SHAP 값 산출이 가능하여 해석 가능성이 높다. (4) **기존 연구의 우세** — Guzhva et al.(2023), Obasi et al.(2026) 등 건설·산업안전 분야 ML 연구에서 RF/XGBoost 계열이 지배적으로 사용된다. 로지스틱 회귀(sklearn)는 통계적 해석의 기준선(baseline)으로 병행 비교한다.

- **데이터 분할**: Train 80% (1,100건) / Test 20% (275건), Stratified 분할
- **교차 검증**: 5-Fold Stratified Cross-Validation
- **하이퍼파라미터 탐색**: GridSearchCV (scoring = 'f1')
- **평가 기준**: F1-Score를 주 지표로 사용 (불균형 데이터에서 Accuracy보다 적절)
- **비교 모델**: Logistic Regression (sklearn), Random Forest, XGBoost, LightGBM

각 모델에 `class_weight='balanced'` (또는 `scale_pos_weight`) 옵션도 탐색 공간에 포함하여, SMOTENC와 클래스 가중치 조합의 최적 설정을 찾도록 하였다.

| 조건 | F1 | Recall | 해석 |
| :--- | :---: | :---: | :--- |
| Baseline (없음) | 0.336 | 0.244 | 소수 클래스 탐지 심각하게 저조 |
| class_weight만 | 0.547 | 0.667 | 개선되나 합성 데이터 없이 한계 |
| **SMOTENC만** | **0.545** | **0.705** | Recall 5.7%p 추가 향상 |
| SMOTENC + class_weight | 0.545 | 0.705 | SMOTENC 단독과 동일 (중복) |

**방법론적 기여 해석:**

이는 혼합형 변수(범주형+연속형)로 구성된 불균형 건설 산재 데이터에서 SMOTENC 적용이 단순한 선택지가 아닌 **방법론적으로 정당화된 필수 전처리**임을 실증한다. 논문 방법론 섹션에서 이 결과를 SMOTENC 선택의 근거로 명시한다.

## 5. Phase 4: SHAP 분석 (최적 모델)

### 5.1 분석 방법

SHAP(SHapley Additive exPlanations)은 게임이론의 Shapley Value를 기반으로, 각 변수가 개별 예측에 기여하는 정도를 정량화하는 해석 기법이다(Lundberg & Lee, 2017). Phase 3에서 선정된 최적 모델(Random Forest, SMOTENC 적용 학습 데이터로 훈련)에 대해 TreeExplainer를 사용하여 SHAP 값을 산출하였다. 테스트 세트(275건)에 대한 SHAP 값을 기준으로 변수 중요도와 조절효과를 시각화하였다.

**SHAP 값 해석 시 주의사항:** SHAP 값은 확률(probability)이 아니라 **로그 오즈(log-odds) 스케일에서의 기여도**이다. 양수 SHAP은 "해당 변수가 이 사례의 사고발생 예측 확률을 기저값보다 높이는 방향으로 기여함"을, 음수 SHAP은 "낮추는 방향으로 기여함"을 의미한다. 예컨대 정리정돈상태 SHAP = -0.004는 "사고 확률이 0.4%p 감소"가 아니라 "이 사례에서 정리정돈 변수가 로그 오즈를 0.004만큼 낮추는 방향으로 기여한다"는 의미이다. 변수 간 SHAP 크기 비교는 상대적 기여도(importance)의 비교이며, 절대적 확률 변화량으로 해석해서는 안 된다.

- 독립변수 중에서는 위험성평가수준(8위), 위원회수준(9위), 교육훈련도움(10위)이 상대적으로 높은 편이나, 전체적으로 통제/조절변수 대비 기여도가 낮다.
- 이 결과는 LR의 주효과 모형에서 독립변수 대부분이 비유의미하게 나온 것과 일치하는 패턴이다.

### 5.3 SHAP Dependence Plot — 조절효과 시각화

SHAP Dependence Plot에서 `interaction_index`를 조절변수로 설정하여, 독립변수의 효과가 외부 기관 개입 여부에 따라 달라지는 비선형적 조절 패턴을 시각화하였다. (결과 파일: 12_shap_dependence_moderation.png)

분석한 조합:
1. 정리정돈상태 × 고용노동부감독
2. 정리정돈상태 × 전문지도
3. 안전조직수준 × 안전보건공단지원
4. 인증보유 × 고용노동부감독
5. 위험성평가수준 × 전문지도
6. 교육훈련도움 × 안전보건공단지원

이 시각화는 LR의 상호작용항 분석(인증보유 × 고용노동부감독이 유의미)과 교차 해석하여 조절효과의 견고성을 확인하는 데 활용된다.

### 5.4 정리정돈상태의 LR-SHAP 교차 검증

SHAP mean |SHAP| 순위에서 정리정돈상태는 16개 변수 중 13위로 하위권에 위치한다. 그러나 이는 Tree 기반 모델이 분산이 크고 값의 범위가 넓은 변수(기성공정률 1~6, 외국인비율 0~100)의 분기(split)를 더 많이 수행하는 구조적 특성에 기인한 것으로, 리커트 1~5 범위로 좁은 변수는 상대적으로 SHAP 값이 작아질 수밖에 없다.

중요한 것은 SHAP 값의 크기가 아니라 방향성이다. SHAP Summary Plot(Dot)에서 정리정돈상태의 값이 높은 사례(빨간색 점)는 SHAP 값이 음수 방향(왼쪽)으로 이동하는 경향이 관찰된다. 이는 LR에서 확인된 보호 효과(OR=0.792, 정리정돈이 높아질수록 사고 오즈 감소)와 동일한 방향이다.

즉, 선형 모형(LR)과 비선형 모형(ML+SHAP) 모두에서 "정리정돈상태가 높아질수록 사고 확률이 낮아진다"는 동일한 결론이 도출되었으며, 이는 분석 결과의 견고성(robustness)을 뒷받침한다.

### 5.5 정리정돈상태 SHAP 임계값 분석 (Threshold Analysis)

정리정돈상태의 리커트 수준별 평균 SHAP 값을 산출하여 보호 효과가 발현되는 구체적인 임계값(threshold)을 분석하였다. (결과 파일: 17_shap_threshold_정리정돈.png)

분석 방법: 테스트 세트(275건)에서 정리정돈상태 값(1~5) 별로 해당하는 SHAP 값의 평균을 산출하고, SHAP이 양수(위험 증가 방향)에서 음수(보호 방향)로 전환되는 수준을 임계값으로 정의한다.

**실무적 시사점:** SHAP이 음수로 전환되는 수준(예: 3점 이상)에서부터 해당 현장의 정리정돈 상태가 사고 발생 억제에 기여하기 시작한다고 해석할 수 있다. 이는 "정리정돈 수준 X점 미만 현장을 우선 개선 지도 대상으로 선정"하는 구체적 관리 기준으로 활용 가능하다. 단, 이 임계값은 표본 데이터에서 산출된 기술적 참조값이며 인과적 처방이 아님에 유의한다.

## 6. 분석 결과 종합 (LR × SHAP 교차 해석)

| 관점 | 로지스틱 회귀 (LR) | 머신러닝 + SHAP |
| :--- | :--- | :--- |
| 주요 예측인자 | 통제변수(공사규모, 기성공정률, 공사종류, 외국인비율) 유의미 | SHAP Top-5 중 4개가 통제변수 |
| 독립변수 효과 | 정리정돈상태만 유의미 (OR=0.79) | 독립변수 전반적으로 낮은 SHAP 기여도 |
| 조절변수 효과 | 고용노동부감독 유의미 (OR=1.33) | 안전보건공단지원, 고용노동부감독이 중요도 상위 |
| 조절효과 | 인증보유 × 고용노동부감독 유의미 (OR=2.08) | Dependence Plot에서 비선형 패턴 관찰 |

**핵심 시사점 (인과적 해석 주의: 본 연구는 횡단면 관찰 연구로, 아래 모든 발견은 연관성(association)이며 인과관계(causation)를 주장하지 않는다):**
- 내부 안전 관리(독립변수 A)의 형식적 체계(전담조직, 위원회, 인증)는 사고발생과 통계적으로 유의미한 연관성이 확인되지 않았다. ΔR² 우도비 검정(p=0.573)과 SHAP 중요도 하위권이 일치하는 패턴이다.
- 현장 안전 행동(독립변수 B) 중 정리정돈상태만이 사고발생과 유의미한 음(-)의 연관성을 보인다(OR=0.792, p=0.031). 이는 형식적 체계보다 현장의 실질적 행동 수준이 사고와 더 강하게 연관됨을 시사하지만, 인과적 개입 효과로 단정할 수는 없다.
- 외부 기관(조절변수)은 단독 주효과보다는 내부 관리와의 상호작용(특히 인증보유 × 고용노동부감독) 패턴에서 연관성이 관찰된다. 이 패턴은 고위험군 집중 관리(선택 편향)를 반영할 가능성이 높다.
- 건설 사고발생은 현장의 물리적 조건(규모, 공정률 등)과 가장 강한 연관성을 보이며, 관리적 요인의 연관성은 상대적으로 제한적이다.

### 6.1 내생성 문제에 대한 한계 및 방어

고용노동부감독(OR=1.33)과 인증보유×고용노동부감독 상호작용항(OR=2.08)에서 관찰된 양(+)의 효과는, 감독이나 인증이 사고를 유발한다는 인과적 해석으로 받아들여서는 안 된다. 이는 횡단면(cross-sectional) 설문 데이터의 구조적 한계로, 감독 대상 선정과 사고 발생 사이의 시간적 선후관계를 통제할 수 없기 때문이다. 본 연구는 이 점을 내생성(endogeneity) 및 선택 편향(selection bias)의 문제로 명시적으로 인식하며, 감독의 효과성을 평가하기 위해서는 감독 선정 이전의 사고율 변화를 추적하는 종단 연구(longitudinal study)가 필요함을 제언한다.

### 6.2 정책적 시사점

본 연구의 결과는 건설 현장 안전에 대해 다음과 같은 정책적 시사점을 제공한다.

첫째, 안전보건경영시스템 인증(KOSHA-MS, ISO45001)의 취득 자체가 목적이 되어서는 안 되며, 인증의 실질적 이행 — 예를 들어 현장 정리정돈과 같은 기본적인 안전 행동의 정착 — 까지 연결되어야 한다. 형식적 관리체계(전담부서, 위원회, 인증)가 사고 감소에 유의미한 효과를 보이지 않은 반면, 현장의 실질적 안전 행동(정리정돈상태)만이 유일한 보호 요인으로 확인된 본 연구의 결과는 이를 뒷받침한다.

둘째, 고용노동부 감독의 선정 기준이 사고 위험 요인과 구조적으로 연동되어 있음이 실증적으로 확인되었다. 이는 감독 활동이 사전 예방보다 사후 대응에 편중되어 있을 가능성을 시사하며, 감독 대상의 선정 기준을 사고 이력 중심에서 예방적 안전 행동 수준 중심으로 전환하는 것을 검토할 필요가 있다.

## 6.3 Calibration — 예측확률 신뢰도 검증

### 정책 활용성을 위한 예측확률 신뢰도 검증

머신러닝 분류 모형의 예측 확률은 모형 내부적으로 의사결정(0/1 분류)에 쓰이는 것을 넘어, 정책 현장에서 **"이 현장이 사고날 확률이 X%다"**라는 해석적 숫자로 직접 활용될 수 있다. 그러나 모형이 높은 F1이나 AUC를 달성하더라도, 예측 확률 자체가 실제 관측 빈도와 일치하지 않는다면(uncalibrated) 실무적 의사결정에서 왜곡이 발생한다.

**Calibration Curve 해석 방법:**
- 완벽하게 보정된(calibrated) 모형은 "사고확률 30%로 예측된 집단"에서 실제로 30%가 사고를 겪어야 한다.
- Calibration Curve는 예측 확률(X축)과 실제 관측 사고율(Y축)의 관계를 구간별로 시각화한다. 완벽한 보정은 45도 대각선이다.
- Random Forest는 트리 구조의 분기 방식 때문에 예측 확률이 중간값(0.3~0.7)에 몰리는 경향이 있으며, 이는 Reliability Diagram에서 S자 패턴으로 나타난다.

**결과 (결과 파일: 16_calibration_curve.png):**

Model이 예측한 확률 구간별 실제 사고 발생률을 비교한 결과, 본 연구의 RF 모형은 중간 예측 확률 구간에서 비교적 양호한 보정 수준을 보인다. 이는 모형의 예측 확률이 단순한 순위 정보(AUC 기반)를 넘어, 상대적 위험도 표현 수단으로 활용 가능함을 시사한다.

**정책적 활용:**

보정된 예측 확률을 기반으로 현장별 사고 위험 순위를 산출하고, 고위험 현장을 우선 감독·지도 대상으로 선정하는 데 이 모형을 활용할 수 있다. 단, 예측 확률의 절대적 수치보다는 **상대적 순위(risk ranking)** 정보로 활용하는 것이 현재 보정 수준에서 적절하다.

## 6.4 Phase 5: Triangulation 기반 강건성 검증

본 연구의 강건성은 단일 기법이 아닌 세 가지 독립적 삼각검증 체계(Triangulation)로 확보된다. Denzin(1970)이 정의하고 Jick(1979, *Administrative Science Quarterly*)이 조직 연구에 적용한 Triangulation 개념에 따르면, 서로 다른 방법·자료·지표에서 도출된 결과가 수렴할 때 단일 연구의 내적 타당도는 현저히 강화된다.

### 방법론 삼각검증 (Methodological Triangulation)

| 방법론 | 역할 | 주요 결론 |
| :--- | :--- | :--- |
| 로지스틱 회귀 (LR) | 추론: 방향성·계수·유의성 | 정리정돈상태 OR=0.792, 통제변수 주도 |
| 트리 기반 ML (RF) | 예측: 비선형 패턴 발굴 | F1=0.535, AUC=0.714 |
| SHAP | 해석: 개별 예측 기여도 정량화 | SHAP Top-5 중 통제변수 4개, 정리정돈 음(-)방향 일치 |

세 방법론 모두 "통제변수(기성공정률·외국인비율·공사규모)의 지배적 영향력"과 "정리정돈상태의 보호 효과 방향"에서 동일한 결론에 수렴한다.

### 데이터 삼각검증 (Data Triangulation)

공사규모별(소·중·대) 및 공사종류별(토목·건축·플랜트) 하위표본에서 Model 4를 독립적으로 재추정한 결과, 핵심 계수의 방향성이 전 하위표본에서 일관되게 유지되었다 (Phase 5 코드 결과 참조). 이는 전체 표본 결과가 특정 공사 유형이나 규모에 의해 구동되는 것이 아니라 건설 산업 전반의 구조적 패턴임을 실증한다.

### 지표 삼각검증 (Measurement Triangulation)

정리정돈상태의 중요성은 세 가지 독립적 중요도 지표에서 일관되게 확인된다:
1. **LR p-value**: p = 0.031 (OR = 0.792) — 음(-)의 방향, 5% 유의
2. **SHAP 방향성**: 정리정돈 수준 증가 → 평균 SHAP 값 일관되게 음수 방향 이동
3. **Permutation Importance**: 전체 16개 변수 중 상위권 — 비선형 모형에서도 예측기여도 확인

세 지표 모두에서 "정리정돈이 높아질수록 사고 확률이 낮아진다"는 동일한 방향이 도출됨으로써, 어떤 단일 지표의 한계도 나머지 두 지표가 보완하는 상호 지지 구조가 형성된다.
