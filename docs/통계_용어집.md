## 1. 로지스틱 회귀 관련 지표

### 계수 (Coefficient, B)

로지스틱 회귀에서 각 독립변수가 종속변수의 **로그 오즈(log-odds)** 에 미치는 영향의 크기와 방향.

- 양수(+): 해당 변수가 높아질수록 사고 발생 가능성 증가 방향
- 음수(−): 해당 변수가 높아질수록 사고 발생 가능성 감소 방향
- 0에 가까울수록 영향 없음
- 계수 자체는 "로그 오즈"의 변화량이라 직관적이지 않음 → **오즈비(OR)로 변환해서 해석하는 게 일반적**

**숫자 읽는 법:**

| B 값 | 방향 | 크기 판단 |
| :---: | :--- | :--- |
| −0.234 | 음수 → 사고 감소 방향 | 0에 가깝지 않으므로 어느 정도 효과 있음 |
| 0.012 | 양수 → 사고 증가 방향이지만 | 0에 매우 가까워 영향이 거의 없음 |
| 0.733 | 양수 → 사고 증가 방향 | 절댓값이 크므로 효과가 강함 |

> 본 연구 예:  
> `정리정돈상태` B = −0.234 → "정리정돈이 1점 높아질수록 사고 발생 로그 오즈가 0.234 낮아진다"  
> `외국인비율` B = 0.012 → "외국인비율이 1%p 높아질수록 사고 발생 로그 오즈가 0.012 높아진다" (작은 수치이지만 외국인비율 범위가 0~100%이므로 누적 영향은 상당할 수 있음)

---

### 오즈비 (Odds Ratio, OR)

계수(B)를 지수변환(exp(B))한 값. 독립변수가 1단위 증가할 때 사고발생 **오즈가 기존 대비 몇 배**가 되는지를 나타낸다.  
**오즈(Odds)**란 사고가 발생할 확률을 발생하지 않을 확률로 나눈 값이다.

**숫자 읽는 법:**

| OR 값 | 해석 방법 |
| :---: | :--- |
| OR = 1.000 | 영향 없음 (기준선) |
| OR = 1.334 | 해당 조건일 때 사고 오즈가 **33.4% 증가** (1.334 − 1 = 0.334, 즉 +33.4%) |
| OR = 1.492 | 해당 조건일 때 사고 오즈가 **49.2% 증가** |
| OR = 2.081 | 해당 조건일 때 사고 오즈가 **2.08배** (즉 108% 증가) |
| OR = 0.792 | 해당 조건일 때 사고 오즈가 **20.8% 감소** (1 − 0.792 = 0.208) |
| OR = 0.858 | 해당 조건일 때 사고 오즈가 **14.2% 감소** |
| OR = 1.012 | 해당 조건일 때 사고 오즈가 **1.2% 증가** (수치 자체는 작음) |

> "OR = 1.012가 너무 작아서 의미 없는 것 아닌가?" → 변수의 단위를 함께 봐야 한다.  
> `외국인비율` OR = 1.012는 "1%p 증가 시 1.2% 증가"이지만, 외국인비율이 0%에서 50%로 오르면 OR^50 = 1.012^50 ≈ 1.82, 즉 82% 증가로 누적 효과가 크다.

**계산법 요약:**
- OR > 1: `(OR − 1) × 100`% 증가 → 예: OR = 1.334 → 33.4% 증가
- OR < 1: `(1 − OR) × 100`% 감소 → 예: OR = 0.792 → 20.8% 감소
- OR ≈ 2: "약 2배 높다"고 표현 가능

---

### 95% 신뢰구간 (95% Confidence Interval, 95% CI)

오즈비 추정치의 불확실성 범위. "같은 방식으로 반복 연구한다면 100번 중 95번은 이 구간 안에 참값이 있다"는 의미.

**숫자 읽는 법: 구간이 1을 포함하는지가 핵심이다.**

| CI 예시 | 1 포함 여부 | 판단 |
| :---: | :---: | :--- |
| [0.640, 0.979] | 포함 안 함 (전부 1 미만) | OR이 통계적으로 유의미 → 보호효과 있음 |
| [1.012, 1.758] | 포함 안 함 (전부 1 초과) | OR이 통계적으로 유의미 → 위험 증가 효과 있음 |
| [0.729, 1.307] | **1을 포함** | OR이 유의미하지 않음 → 우연일 가능성 배제 불가 |
| [1.005, 1.019] | 포함 안 함 (전부 1 초과) | 구간이 좁고 1보다 작은 마진으로 유의미 |

> 구간이 넓을수록 추정의 불확실성이 크고, 좁을수록 정밀한 추정이다.  
> 예: `외국인비율` CI = [1.005, 1.019] → 구간이 매우 좁고 1 초과 → 유의미하며 신뢰도 높음

---

### p-value (유의확률)

귀무가설("이 변수는 사고발생에 영향이 없다")이 참일 때, 지금 이 데이터처럼 극단적인 결과가 우연히 나올 확률.  
**p가 작을수록 우연이 아닐 가능성이 높다 = 통계적으로 유의미하다.**

**숫자 읽는 법:**

| p-value 예시 | 유의성 표기 | 해석 |
| :---: | :---: | :--- |
| p = 0.000 | *** | 0.1% 미만 확률로 우연 → 매우 강한 유의미성 |
| p = 0.001 | ** | 0.1% 수준 → 강한 유의미성 |
| p = 0.022 | * | 2.2%로 우연일 가능성 → 5% 기준 유의미 |
| p = 0.031 | * | 3.1% → 유의미 |
| p = 0.038 | * | 3.8% → 유의미 (5%에 근접하지만 기준 이하) |
| p = 0.041 | * | 4.1% → 유의미 |
| p = 0.118 | (없음) | 11.8%로 우연일 수 있음 → 유의미하지 않음 |
| p = 0.160 | (없음) | 16%로 우연 가능성 높음 |
| p = 0.274 | (없음) | 27.4% → 유의미하지 않음 |
| p = 0.573 | (없음) | 57.3% → 사실상 영향 없음 |
| p = 0.871 | (없음) | 우연일 가능성이 매우 높음 |
| p = 0.994 | (없음) | 거의 완전히 우연 수준 |

> p = 0.000 은 "정확히 0"이 아니라 "소수점 셋째 자리 이하로 매우 작음"을 의미 (컴퓨터 출력상 반올림)  
> **p-value는 효과 크기가 아니다**: p가 0.001이라도 OR이 1.01이면 효과가 매우 작다. OR과 CI를 함께 봐야 한다.

## 2. 모형 적합도 지표

### Pseudo R² (맥패든 R², McFadden's R²)

로지스틱 회귀에서 선형 회귀의 결정계수(R²)에 해당하는 지표. 모형이 종속변수를 얼마나 잘 설명하는지를 0~1 사이로 나타낸다.

**숫자 읽는 법:**

| Pseudo R² 값 | 해석 |
| :---: | :--- |
| 0.00 ~ 0.05 | 거의 설명력 없음 |
| 0.05 ~ 0.10 | 낮은 설명력 (사회과학 설문데이터에서는 흔한 수준) |
| 0.10 ~ 0.20 | 보통 수준 |
| 0.20 ~ 0.40 | 충분한 적합 |
| 0.40 이상 | 우수한 적합 |

> 중요: 선형 회귀의 R² = 0.10이면 "10% 설명"으로 낮지만,  
> 로지스틱의 Pseudo R² = 0.10은 같은 기준으로 보면 안 된다. 사회과학 분야 설문 이진분류에서 0.10~0.15는 일반적으로 수용 가능한 수준이다.

> 본 연구 예:  
> Model 1 (통제변수만): 0.0927 → 현장 물리적 특성만으로도 약 9.3% 설명  
> Model 4 (전체 주효과): 0.1044 → 관리/행동 변수 추가로 10.4%로 소폭 상승  
> Model 5 (상호작용 포함): 0.1212 → 12.1%이지만 BIC 악화 → 과적합 의심

---

### AIC (Akaike Information Criterion)

모형의 설명력과 복잡도(변수 수) 사이의 균형 지표.

**숫자 읽는 법:**
- **절대값 자체는 의미 없다**. 반드시 **같은 데이터**에서 비교한 두 모형 간 차이로 판단한다.
- **낮을수록 좋다**. 변수를 추가할 때 AIC가 내려가면 → 그 변수는 추가할 가치가 있다.
- 변수를 추가해도 AIC가 그대로이거나 올라가면 → 그 변수는 설명력보다 복잡도가 더 늘어난 것.

| 비교 예시 | AIC | 판단 |
| :--- | :---: | :--- |
| Model 1 (통제 5개) | 1501.7 | 기준 |
| Model 2 (+독립A 3개) | 1505.7 | AIC **상승** → 독립A 추가 효과 미미 |
| Model 4 (+조절 3개) | 1504.5 | AIC **하락** → 조절변수 추가 의미 있음 |

---

### BIC (Bayesian Information Criterion)

AIC와 같이 낮을수록 좋지만, 변수 추가에 더 강한 패널티를 부여한다 (표본 크기 n을 추가 반영).

**숫자 읽는 법:**
- AIC와 동일하게 **낮을수록 좋고, 모형 간 비교**에 사용
- AIC보다 변수 추가에 엄격 → BIC가 AIC와 반대 방향을 가리키면 BIC를 우선

> 본 연구 Model 5: AIC = 1524.8, BIC = 1739.1  
> Model 4 대비 AIC는 상승, BIC도 크게 상승 → 상호작용항 24개 추가가 복잡도 대비 이득 없음 → 과적합

---

### Log-Likelihood (로그우도, ln L)

모형이 현재 데이터를 얼마나 잘 설명하는지를 나타내는 기반 값. 항상 **음수**이며 0에 가까울수록 좋다.

**숫자 읽는 법:**
- **절대값 자체보다 차이가 중요하다**. 두 모형의 Log-Likelihood 차이로 LR 통계량을 계산한다.
- 더 많은 변수를 넣으면 Log-Likelihood는 항상 0에 가까워진다(개선). 문제는 그 개선이 충분한가.

| 모형 | Log-Likelihood | 의미 |
| :--- | :---: | :--- |
| Model 1 | −744.8 | 기준 |
| Model 4 | −735.2 | 9.6 개선 (= 변수 추가가 설명력 향상) |
| Model 5 | −721.4 | 13.8 추가 개선 (하지만 24개 변수 추가치고는 작음) |

---

## 3. 가설 검정

### LR 통계량 (Likelihood Ratio Statistic, 우도비 통계량)

두 모형의 Log-Likelihood 차이를 이용해 "변수 그룹 추가가 유의미한 개선인가"를 검정하는 통계량.

$$LR = -2 \times (\ln L_{이전 모형} - \ln L_{이후 모형})$$

**숫자 읽는 법:**
- LR 통계량이 클수록 두 모형 간 설명력 차이가 크다.
- 이 값이 카이제곱($\chi^2$) 분포를 따르는지를 df와 함께 보고, p-value로 유의미성을 판단한다.
- **LR 단독으로 판단 불가** → 반드시 df와 p-value를 함께 봐야 한다.

| 비교 | LR | df | p-value | 판단 |
| :--- | :---: | :---: | :---: | :--- |
| M1→M2 (독립A 추가) | 2.00 | 3 | 0.573 | 유의미하지 않음. 변수 3개를 추가했는데 설명력 향상이 우연 수준 |
| M3→M4 (조절변수 추가) | 8.40 | 3 | 0.038 | 유의미 (5% 기준). 3개 변수 추가로 유의미한 설명력 향상 |
| M4→M5 (상호작용 24개) | 27.65 | 24 | 0.274 | 유의미하지 않음. 24개 추가 대비 향상 불충분 |

> LR = 2.00이라도 df = 1이면 p가 낮을 수 있고, LR = 27.65도 df = 24이면 p가 높을 수 있다.  
> "큰 LR = 유의미"가 아니라, df를 감안한 p-value가 기준이다.

---

### df (Degrees of Freedom, 자유도)

LR 검정에서 추가된 파라미터(변수)의 수. 카이제곱 분포의 모양을 결정하여 p-value 계산에 영향을 준다.

**숫자 읽는 법:**
- df = 추가한 변수의 개수
- df가 클수록 같은 LR 통계량에서 p-value가 높아진다 (= 변수를 많이 추가하면 유의미해지기 어려워짐)

> 예: LR = 8.40, df = 3 → p = 0.038 (유의미)  
> 만약 같은 LR = 8.40이지만 df = 10이었다면 → p ≈ 0.59 (유의미하지 않음)  
> → 변수를 한꺼번에 많이 추가하면 개별 효과가 희석된다

## 4. 머신러닝 성능 지표

### Accuracy (정확도)

전체 예측 중 맞춘 비율.

**숫자 읽는 법:**

| Accuracy | 주의사항 |
| :---: | :--- |
| 0.658 | 65.8% 정확. 불균형 데이터에서는 이 숫자를 맹신하면 안 됨 |

> 본 연구에서 사고 미발생이 71.6%이므로, 모든 사업장을 "사고 없음"으로 예측해도 Accuracy = 71.6%가 된다.  
> Accuracy = 0.66인 모델이 오히려 실제 사고를 탐지하고 있는 것.  
> **불균형 데이터에서는 F1-Score와 Recall을 우선 본다.**

---

### Precision (정밀도)

"사고 발생이라고 예측한 것 중 실제로 사고가 난 비율"

**숫자 읽는 법:**

| Precision | 해석 |
| :---: | :--- |
| 0.436 | 사고 발생 예측 10건 중 약 4.4건만 실제 사고. 오탐이 많음 |
| 0.84 | 사고 없음 예측 시 84%가 실제 사고 없음 |

---

### Recall (재현율)

"실제로 사고가 난 사업장 중 모델이 탐지해낸 비율"

**숫자 읽는 법:**

| Recall | 해석 |
| :---: | :--- |
| 0.692 | 실제 사고 사업장 중 69.2%를 탐지함. 나머지 30.8%는 미탐 |
| 0.000 | 사고 발생 사업장을 하나도 탐지 못함 (SMOTENC 적용 전 LightGBM) |

> 안전 분야에서 Recall이 낮다는 것은 "실제 위험한 현장을 위험하지 않다고 판단"하는 것이므로 매우 큰 문제.  
> Precision vs Recall: Precision을 올리면 Recall이 내려가고, Recall을 올리면 Precision이 내려가는 트레이드오프 관계.

---

### F1-Score

Precision과 Recall의 조화 평균. 둘 다 반영한 종합 지표.

**숫자 읽는 법:**

| F1 | 해석 |
| :---: | :--- |
| 0.535 | 전반적인 탐지 능력이 약 53.5% 수준 |
| 0.000 | Precision이나 Recall 중 하나가 0 → 사실상 탐지 불가 |

> F1 = 0.535는 낮아 보이지만, 설문 기반 이진분류에서 사회과학적으로 수용 가능한 수준.  
> 임상/의료 분야에서는 F1 ≥ 0.8 이상을 요구하는 경우가 많으나, 간접 설문 데이터는 예측 자체가 어렵다.

---

### ROC-AUC

모든 분류 임계값에서 모델의 탐지 능력을 종합한 지표.

**숫자 읽는 법:**

| AUC | 해석 |
| :---: | :--- |
| 0.5 | 동전 던지기와 동일 (무의미) |
| 0.692 | 보통 수준 (Logistic Regression) |
| 0.714 | 양호 수준 (Random Forest 최적 모델) |
| 0.710 | 양호 수준 (XGBoost) |

> AUC = 0.714는 "임의로 선택한 사고발생 사업장과 미발생 사업장 쌍에서 71.4% 확률로 모델이 올바르게 구분한다"는 의미.

## 5. 전처리 관련 용어

### VIF (Variance Inflation Factor, 분산팽창지수)

독립변수 간 다중공선성(서로 너무 비슷한 변수끼리 겹치는 문제) 수준을 측정하는 지표.

**숫자 읽는 법:**

| VIF | 판단 |
| :---: | :--- |
| 1.0 | 완전 독립 (이상적) |
| 1.047 ~ 1.761 | 본 연구 범위. 매우 낮음 → 다중공선성 없음 |
| 5 이상 | 주의 필요 |
| 10 이상 | 심각. 해당 변수 제거 또는 변환 필요 |

> VIF = 1.761 (교육훈련도움): "이 변수의 분산이 다른 변수들로 인해 1.761배 팽창되어 있다"  
> 10 미만이므로 전혀 문제없는 수준.

---

### SMOTENC (Synthetic Minority Over-sampling Technique for Nominal and Continuous)

클래스 불균형(사고발생 28.4% vs 미발생 71.6%) 문제 해결을 위해 소수 클래스의 **인공 합성 데이터**를 생성하는 기법.

- 일반 SMOTE는 연속형 변수에만 적합 → 범주형 변수에서 0.4, 1.7 같은 비현실적 값 생성
- SMOTENC는 범주형 변수에 **최빈값(mode) 기반 합성**으로 현실적 값 유지
- **반드시 교차검증 fold 안에서만 적용** → 외부 적용 시 데이터 누수 발생

---

### Listwise Deletion (완전사례분석)

분석 변수 중 하나라도 결측치가 있는 사례 전체를 제외하는 방법.

- 1,502 → 1,375로 감소한 원인
- 대체(imputation) 없이 제외하므로 표본 일관성 보장, 단 표본 손실 발생

---

### 역인과성 (Reverse Causality) / 선택 편향 (Selection Bias)

인과 방향이 실제와 반대로 관측되는 문제.

**숫자 읽는 법 (해석 주의 사례):**

> `고용노동부감독` OR = 1.334, p = 0.041  
> 단순 해석: "감독을 받으면 사고가 33.4% 더 많이 발생한다"  
> 올바른 해석: "사고 위험이 높은 사업장이 감독 대상으로 선정되므로, 감독을 받은 집단이 원래부터 위험한 집단이다"  
> → OR이 양수라고 해서 "감독 = 사고 증가 원인"으로 해석하면 안 됨

---

## 6. SHAP 관련

### SHAP Value (SHapley Additive exPlanations)

각 변수가 개별 예측값에 얼마나 기여했는지를 정량화. 게임이론의 Shapley Value 기반.

**숫자 읽는 법:**

| SHAP 값 | 해석 |
| :---: | :--- |
| 양수 (+) | 이 변수가 해당 사업장의 사고발생 예측확률을 **높이는** 방향으로 기여 |
| 음수 (−) | 이 변수가 해당 사업장의 사고발생 예측확률을 **낮추는** 방향으로 기여 |
| 0에 가까움 | 이 사업장의 예측에 해당 변수의 영향이 거의 없음 |

> mean |SHAP|: 테스트 세트 전체의 SHAP 절댓값 평균 → 변수 중요도(Feature Importance)  
> 방향(+/-)에 관계없이 절댓값이 클수록 중요한 변수

---

## 7. 기타 개념

### 내생성 (Endogeneity)

독립변수와 오차항 사이에 상관관계가 있어, 회귀 추정치가 편향되는 문제. 역인과성·누락변수가 주요 원인.

---

### 조절효과 (Moderation Effect)

독립변수(X)가 종속변수(Y)에 미치는 영향이 조절변수(M)의 수준에 따라 달라지는 현상.

**숫자 읽는 법:**

> `인증보유 × 고용노동부감독` 상호작용항: 계수 = 0.733, OR = 2.081, p = 0.022  
> 해석: "인증을 보유하고 감독도 받은 사업장은, 두 요인의 주효과를 합산한 것보다 사고 오즈가 추가로 2.08배 높아진다"  
> → 인증과 감독의 결합이 단순 합산보다 강한 (양의) 상호작용 효과를 가짐

---

### 계층적 회귀 (Hierarchical Regression)

독립변수를 이론적 순서에 따라 단계적으로 투입하는 방법. 각 그룹의 순수 설명력 기여를 분리 확인.

**숫자 읽는 법 (ΔR² 해석):**

| 비교 | ΔPseudo R² | 해석 |
| :--- | :---: | :--- |
| M1→M2: 독립A 추가 | +0.0012 | 내부 관리 변수 3개가 설명력을 0.12%p 올림 → 매우 미미 |
| M2→M3: 독립B 추가 | +0.0053 | 안전행동 변수 5개가 0.53%p 올림 → 소폭 기여 |
| M3→M4: 조절변수 추가 | +0.0052 | 외부기관 변수 3개가 0.52%p 올림 (p=0.038로 유일하게 유의미) |

> ΔR² 숫자 자체는 작아 보여도, 이미 통제변수로 설명력이 어느 정도 확보된 상태에서의 추가 기여이므로 의미 있다.

---

## 8. 모형 검증 방법

### Hosmer-Lemeshow 검정 (H-L Test)

로지스틱 회귀 모형이 실제 데이터에 적합한지 확인하는 적합도 검정. 예측 확률을 10개 구간(분위)으로 나누어 각 구간의 예측 빈도와 실제 관측 빈도가 일치하는지를 카이제곱 통계량으로 검정한다.

**숫자 읽는 법:**

| p-value | 판단 |
| :---: | :--- |
| p > 0.05 | 귀무가설("모형이 데이터에 적합하다") 기각 불가 → **모형 적합** |
| p ≤ 0.05 | 예측값과 실제값 사이에 유의미한 불일치 → 모형 개선 필요 |

> 본 연구 Model 4: p = 0.2017 > 0.05 → 적합 판정  
> 주의: H-L 검정은 표본이 클수록 민감해지므로 p-value 단독보다 Pseudo R²·AIC와 함께 해석한다.

---

### FDR 보정 (False Discovery Rate, Benjamini-Hochberg)

다수의 변수를 동시에 검정할 때 우연히 유의미한 결과가 나올 확률(다중비교 문제)을 통제하는 방법. Bonferroni 보정보다 덜 보수적이어서 검정력 손실이 적다.

**숫자 읽는 법:**

- 원본 p-value를 변수 수(n)와 순위(k)를 고려해 조정: `p_FDR = p_원본 × n / k`
- p_FDR < 0.05이면 보정 후에도 유의미, 0.05 이상이면 보정 후 탈락

| 변수 | p_원본 | p_FDR | 판단 |
| :--- | :---: | :---: | :--- |
| 기성공정률 | 0.000 | 0.000 | 보정 후에도 유의미 |
| **정리정돈상태** | **0.031** | **0.098** | 보정 후 5% 탈락, 10% 경향 유지 |
| 고용노동부감독 | 0.041 | 0.109 | 보정 후 비유의미 |

> p_FDR이 0.05를 넘어도 "효과 없음"으로 단정하면 안 된다. 탐색적 연구에서는 보정 전·후를 병기하고 다른 방법론(SHAP, Permutation Importance)과 교차 해석한다.

---

### 표준화 OR (Standardized Odds Ratio)

변수마다 단위가 달라 비표준화 OR 간 직접 비교가 불가능한 문제를 해결하기 위해, 연속·순서·리커트형 변수를 Z-score(μ=0, σ=1)로 표준화한 후 로지스틱 회귀를 재적합하여 산출한 OR.

**적용 범위:**
- **표준화 적용**: 연속·순서형·리커트형 변수 (1 표준편차 증가 시 오즈 변화로 해석)
- **표준화 제외**: 이진변수(인증보유·전문지도·고용노동부감독·안전보건공단지원) — 0→1 변화가 이미 직관적이므로 표준화하면 해석이 불가능해짐

> 표준화 OR이 클수록 해당 변수의 단위 불문 효과 크기가 큰 것. 통제변수(기성공정률, 외국인비율 등)의 표준화 OR이 관리·행동 변수보다 일관되게 높게 나타나 현장 물리적 조건의 우세를 재확인.

---

### Bootstrap 95% 신뢰구간 (Bootstrap Confidence Interval)

단일 테스트 세트 기반 성능 지표는 표본 변동에 민감하다. Bootstrap은 테스트 세트에서 복원 추출(n=1,000회)을 반복하여 성능 지표의 분포를 추정하고, 2.5%~97.5% 구간을 95% CI로 산출한다.

**숫자 읽는 법:**

| F1 평균 | 95% CI | 판단 |
| :---: | :---: | :--- |
| 0.543 | [0.455, 0.621] | 하한(0.455)이 무작위 예측 수준(≈0.39)을 상회 → 탐지 능력 안정적으로 확인 |

> CI가 넓을수록 표본 의존성이 크고, 좁을수록 추정이 안정적이다.  
> point estimate(단일 값)와 Bootstrap 평균이 유사하면 결과의 신뢰도가 높다는 추가 근거가 된다.

---

### McNemar's Test (맥니마 검정)

두 분류 모델이 같은 테스트 세트에서 예측한 결과를 비교하는 쌍체 검정. F1 차이가 작더라도 예측 패턴 자체가 다른지를 통계적으로 확인한다.

**원리:** 두 모델의 예측이 불일치하는 사례를 기반으로 검정
- b: 모델 A만 정답인 사례 수
- c: 모델 B만 정답인 사례 수
- 통계량: (|b−c|−1)² / (b+c), 카이제곱 분포(df=1)

**숫자 읽는 법:**

| p-value | 판단 |
| :---: | :--- |
| p < 0.05 | 두 모델의 예측 성능 차이가 통계적으로 유의미 |
| p ≥ 0.05 | F1 수치 차이가 있더라도 통계적으로는 동등 |

> 본 연구: RF vs XGBoost, p = 0.0303 → RF의 우위가 통계적으로 유의미하게 확인되어 최적 모델 선정 근거로 활용.

---

### Sensitivity Analysis (민감도 분석, 다중 seed)

분석 결과가 특정 random seed 선택에 의존하는지 검증하는 방법. 동일한 하이퍼파라미터로 서로 다른 seed(예: 42, 123, 456, 789, 2024) 5개를 사용해 반복 실행하고 성능 지표의 평균과 표준편차를 확인한다.

**숫자 읽는 법:**

| 지표 | 평균 | 표준편차 | 판단 |
| :--- | :---: | :---: | :--- |
| F1 | 0.551 | 0.019 | 표준편차가 매우 작음 → seed 의존성 없음, 결과 안정적 |

> 표준편차 0.02 이하면 seed에 따른 변동이 크지 않아 재현 가능한 결과로 볼 수 있다.

---

### Ablation Study (절제 연구)

분석에 사용된 구성 요소(전처리 기법, 모델 옵션 등)의 각 독립적 기여를 확인하기 위해, 한 번에 하나의 요소만 제거·변경하여 성능 변화를 비교하는 실험 설계.

**숫자 읽는 법 (본 연구: SMOTENC vs class_weight 비교):**

| 조건 | F1 | Recall | 판단 |
| :--- | :---: | :---: | :--- |
| Baseline (없음) | 0.336 | 0.244 | 불균형 미처리 시 심각한 성능 저하 |
| class_weight만 | 0.547 | 0.667 | 단독으로도 효과적 |
| SMOTENC만 | 0.545 | 0.705 | Recall에서 class_weight 대비 우위 |
| SMOTENC + class_weight | 0.545 | 0.705 | 동시 적용 시 SMOTENC 단독과 동일 → 효과 중복 |

> Ablation Study의 핵심 목적: "이 방법을 왜 선택했는가"를 데이터로 증명하는 것.  
> 결과가 중복(redundant)이라는 발견도 유의미한 결론이다 — class_weight 추가가 불필요함을 실증적으로 확인.

---

### Permutation Importance (순열 중요도)

특정 변수의 값을 무작위로 섞었을 때 모델 성능(F1 또는 AUC)이 얼마나 하락하는지를 측정하여 변수 중요도를 산출하는 방법. SHAP과 독립적인 방법론으로, 두 결과가 일치하면 변수 중요도의 신뢰도가 높아진다.

**숫자 읽는 법:**

| 결과 | 해석 |
| :--- | :--- |
| Permutation Importance가 높음 | 해당 변수를 제거하면 성능이 크게 하락 → 중요한 변수 |
| Permutation Importance가 낮음 | 변수 제거 시 성능 변화 거의 없음 → 예측에 기여 미미 |

> SHAP × Permutation Importance × LR p-value 삼중 교차검증: 세 방법 모두에서 동일한 방향이 확인되면 해당 변수의 효과가 분석 방법론에 무관하게 일관된다는 강한 근거가 된다.  
> 본 연구: 정리정돈상태가 세 방법 모두에서 사고 감소 방향 일치 → 보호 효과 경향 지지.

---

## 9. 표 읽는 법 종합 예시

아래는 Model 4 결과표를 실제로 읽는 방법이다.

```
| 변수          | 계수(B)  | 오즈비(OR) | 95% CI          | p-value | 유의성 |
|--------------|---------|-----------|-----------------|---------|------|
| 정리정돈상태   | -0.234  | 0.792     | [0.640, 0.979]  | 0.031   | *    |
| 공사규모      |  0.400  | 1.492     | [1.201, 1.854]  | 0.000   | ***  |
| 안전조직수준   | -0.024  | 0.976     | [0.729, 1.307]  | 0.871   |      |
```

**`정리정돈상태` 행 읽기:**
1. B = −0.234 → 음수이므로 정리정돈이 높아질수록 사고 감소 방향
2. OR = 0.792 → 1단위 높아질수록 사고 오즈 20.8% 감소
3. CI = [0.640, 0.979] → 1을 포함하지 않음 → 유의미한 효과
4. p = 0.031 → 3.1%로 우연일 가능성 → 5% 기준 유의미 (*)
5. 종합: "정리정돈 수준이 높을수록 사고 발생 위험이 유의미하게 낮아진다"

**`공사규모` 행 읽기:**
1. B = 0.400 → 양수이므로 규모가 클수록 사고 증가 방향
2. OR = 1.492 → 규모 1단계 증가 시 사고 오즈 49.2% 증가
3. CI = [1.201, 1.854] → 1을 포함하지 않음 → 유의미
4. p = 0.000 → 매우 강한 유의미성 (***)
5. 종합: "공사규모가 클수록 사고 발생 위험이 매우 유의미하게 높아진다"

**`안전조직수준` 행 읽기:**
1. B = −0.024 → 0에 매우 가까움 → 사실상 영향 없음
2. OR = 0.976 → 1에 매우 가까움 → 2.4% 감소에 불과
3. CI = [0.729, 1.307] → 1을 포함함 → 유의미하지 않음
4. p = 0.871 → 87.1%로 우연일 가능성 높음
5. 종합: "안전조직수준은 사고발생에 통계적으로 유의미한 영향이 없다"
