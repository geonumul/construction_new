# 논문 집필 시 낮은 ML 성능 방어 전략 메모

F1=0.545, AUC=0.717의 성능을 방어할 때 떠올릴 체크리스트. 저널 게재 사례(Davoudi Kakhki 2019 *Safety Science*, PLoS One 2025, arXiv 2025 등)가 공통으로 사용한 6가지 전략.

## 6가지 방어 전략

1. **연구 목적을 "예측 정확도"가 아니라 "변수 중요도·해석 발굴"로 프레이밍**
   → Phase 3 도입부에 "ML은 비선형 기여 구조 발굴과 LR 교차검증 도구" 명시

2. **기저선 대비 우수성 강조**
   → PR-AUC(0.460) > 기저율(0.284), Bootstrap CI 하한(0.462) > 우연(0.39)

3. **데이터 본질적 한계를 선제적으로 투명 공개**
   → 설문 자기보고·단면·1,375건·17변수 한계를 Phase 3 도입부와 한계 섹션 양쪽에 명시

4. **방법론적 기여·혁신을 부각**
   → SMOTENC Ablation, LR+ML+SHAP 병행, Swiss Cheese Model 이론 연결

5. **해석(SHAP)에서 나오는 통찰이 진짜 가치**
   → SHAP Top-3 통제변수 발견, 정리정돈 보호효과, 인증×감독 선택편향

6. **다층 교차 검증으로 단일 수치 신뢰도 보강**
   → 방법론(LR·ML·SHAP) + 데이터(Subsample) + 지표(SHAP×PI×LR) 삼중 수렴

## 참조 사례

- **Davoudi Kakhki et al. (2019) *Safety Science*** — 33,000건 농업 산재, 인과 추론이 아닌 변수 중요도 발굴 목적 명시로 게재
- **arXiv 2025 occupational accidents forecasting** — balanced accuracy 0.5 근처 모델도 "naive baseline 대비 우수"로 방어해 게재
- **PLoS One 2025 occupational stress** — 한계를 방법론 섹션에서 선제 노출, 3단계 검증(Holdout + 10-fold CV + external) 수렴으로 신뢰도 확보
- **ETASR 2021 hospital OSH** — 성능 수치보다 "OSH 도메인 DSS 프레임워크 제안"이라는 방법론 혁신을 전면에 둠

## 본문 집필 시 적용 포인트

- **Phase 3 도입부 첫 문단**: 전략 1 + 3 동시 명시
- **Phase 3 결과 해석 말미**: 전략 2 명시 (수치 근거 인용)
- **Phase 4 SHAP 도입부**: 전략 5 명시 (해석이 목적)
- **종합 고찰**: 전략 6 명시 (삼중 수렴)
- **한계 섹션**: 전략 3 재확인 (표본·설문·단면)

## 핵심 방어 문장 예시

> "본 연구에서 ML은 예측 정확도 최적화가 아니라 비선형 변수 기여 구조 발굴과 LR 결과 교차 검증을 목적으로 도입되었다. F1=0.545는 설문 자기보고 데이터·1,375건·17변수·단면 구조에서 기저선(기저율 0.284) 대비 개선된 수준이며, 본 연구의 핵심 기여는 SHAP 기반 변수 해석과 다층 교차 검증에 있다."

## 공통 교훈 한 줄

**수치 자체를 방어하려 하지 말고, "연구의 가치가 수치에 있지 않음"을 구조로 보여라.** 도입부·방법론·해석·한계 모든 섹션에서 일관된 프레이밍이 유지돼야 방어가 완결된다.
